API Reference

Base URL: https://website-login-wyxt.onrender.com

Public Endpoints

Method Endpoint Description Auth
POST /api/login Login — body: { identifier, password, role } None
POST /api/register Register student — body: { full_name, email, phone, password, course, plan } None
POST /api/logout Logout — deletes session token Bearer
GET /api/me Get current user info Bearer
POST /api/enquiry Submit course enquiry None
GET /api/status Health check + table counts None

Admin-Only Endpoints

Method Endpoint Description
GET /api/admin/stats Dashboard stats, recent signups, course breakdown
GET /api/admin/students All students list
PATCH /api/admin/students/<id>/toggle Revoke or restore student access
DELETE /api/admin/students/<id> Permanently delete student
GET /api/admin/enquiries All enquiries list
PATCH /api/admin/enquiries/<id>/status Update enquiry status (new / contacted / enrolled)
GET /api/admin/sessions All active login sessions

Deployment

Two-service architecture — static frontend + Python backend

N
Netlify
Static site hosting + CDN
1. Install Netlify CLI: npm install -g netlify-cli
2. Login: netlify login
3. Deploy: netlify deploy --dir . --prod
4. API calls proxied via netlify.toml
# netlify.toml
[[redirects]]
from = "/api/*"
to = "https://...onrender.com/api/:splat"
status = 200
R
Render
Python Flask backend (free tier)
1. Connect GitHub repo at render.com
2. Build: pip install -r requirements.txt
3. Start: python server.py
4. Auto-redeploy on every push to main
Free tier spins down after 15 min idle — first request takes ~30s to wake up. SQLite data resets on redeploy (auto-seeded on startup).
